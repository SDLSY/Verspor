# 开发验证产物目录

本目录用于保存开发过程中的截图、UI dump、smoke 结果、阶段性验证日志和局部问题排查材料。它是调试归档层，不是正式测试交付层。

## 当前主要分层

- `root_archive/`
  - 收口原先散落在根目录的截图、日志、崩溃日志和数据库快照
- `tool_output/`
  - 需要保留的工具稳定输出，例如 Playwright 导出结果
- `smoke/`、`ui-check/`、`verification/`
  - 当前开发验证主目录
- 其他专题目录
  - 按功能、缺陷、阶段或实验主题保留过程证据

## 使用规则

- 若材料最终要进入测试文档或答辩包，应从本目录筛选后复制到 `test-evidence/` 或 `最终材料/`，不要直接把整个目录当正式材料引用。
- 新增子目录时，优先使用“问题名/任务名/阶段名”命名，避免继续出现 `tmp`、`new`、`截图(1)` 这类命名。
- 仓库里默认只跟踪本 README；本目录下的大多数过程文件属于本地归档，不要求进入版本控制。
