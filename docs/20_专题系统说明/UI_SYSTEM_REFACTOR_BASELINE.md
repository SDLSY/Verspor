# UI 系统重构基线

更新时间：2026-03-10

## 1. 本轮目标
- 收口 Android 主次页面的信息架构，不删功能，只重组层级。
- 用一套统一的设计 token、卡片层级、标题层级和按钮语义替换各页自定义风格。
- 把桌面机器人从“麦克风对话入口”改成“页面说明与状态播报入口”。
- 保持现有业务链路、云端接口和 Fragment/XML 技术栈不变。

## 2. 已完成的结构性调整

### 2.1 壳层与全局 Avatar
- `MainActivity` 已切掉桌面机器人麦克风对话入口。
- 桌面机器人改为两种触发方式：
  - 页面进入时按当前页面和用户状态生成说明并播报。
  - 用户点击机器人时重播或重新生成当前页面说明。
- 机器人不再主动请求麦克风权限，不再走桌面对话收音链。
- Avatar 动作只接收本地语义动作，不接受模型自由协议。

### 2.2 医生页
- 顶部状态区压缩为紧凑状态条，首屏主视觉回到聊天列表。
- 输入区压缩为主控制栏，次级能力全部下沉。
- 多模态、生成功能、推荐动作改为可折叠辅助区。
- 辅助区已整体下沉到输入区之后，不再占用聊天列表上方的首屏高度。
- “开始通话”保留连续半双工；“开始录音”保留一次性转写。
- 已加入冷却窗口，避免 AI 播报内容再次被 IAT 当成输入。

### 2.3 今日页
- 问候、标题、摘要层级统一到新 token。
- 页面文案全面改为资源文件，不再散落硬编码中文。
- 风险、恢复、干预摘要改成总览优先，减少首屏噪音。

### 2.4 设备页 / 我的页 / 趋势页
- 设备页已统一为：状态 Hero + 主动作 + 设备列表 + 高级工具折叠。
- 我的页已统一为：用户 Hero + 云端状态 + 设置列表 + 快捷入口。
- 趋势页与今日页已统一到同一组标题、正文、状态样式。

### 2.5 干预相关次级页
- 放松中心、症状引导、医检分析等页面保留原功能，但统一卡片样式、标题层级和按钮语义。
- 3D 干预中心已统一模型与透明背景，不再使用错误模型资源。

## 3. 设计系统收口

### 3.1 当前统一 token
- 标题层级：`PageTitle / PageSubtitle / SectionTitle / MetricValue / BodyLabel / StatusPill`
- 卡片层级：`Card / Card.Subtle / Card.Inline / Card.HeroSoft`
- 按钮语义：`Primary / Tonal / Outlined / Text / Header`
- 页面间距：沿用 `page_padding_*`、`section_gap*`、`card_padding`、`hero_padding`
- 视觉方向：青蓝冷静医疗感，低噪音背景，不堆叠高饱和强调色

### 3.2 当前未做的事
- 不引入 Compose。
- 不做全局视觉大改版，不重建底部导航交互模型。
- 不改任何后端接口或数据契约。

## 4. 桌面机器人当前行为定义
- 不再承担自由语音问答入口。
- 只承担：
  - 页面说明
  - 当前风险/任务状态提示
  - 下一步引导
  - 与当前页面一致的 TTS 播报
- 默认模型路由：
  - `Spark Lite` 生成页面说明
  - 高风险或长上下文可升级 `Spark X`
  - 失败时回退到本地规则脚本
- 默认音色：`x4_lingxiaoyao_em`

## 5. 医生语音模式当前定义
- 通话模式固定为半双工连续：
  - 聆听中 -> 识别中 -> 等待回复 -> 播报中 -> 冷却 -> 恢复聆听
- TTS 播报期间绝不启用 IAT。
- TTS 结束后增加固定冷却窗口，再恢复监听。
- 进入通话时若已有播报在进行，会先强制停止现有播报。
- 所有 TTS 默认音色统一为 `x4_lingxiaoyao_em`。

## 6. 仍属下一轮的 UI 债
- 医生页辅助区还可继续拆成底部抽屉，而不是同页折叠区。
- 首页与趋势页还可以进一步抽成复用 section include，而不是沿用现有大布局。
- 仍有部分次级页面沿用旧 View 组合，没有完全抽成 `UiBinder / Renderer`。
- 这轮重点是“统一骨架和页面主次”，不是“彻底组件化重写”。
