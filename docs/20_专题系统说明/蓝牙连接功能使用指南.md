## 📱 如何使用蓝牙功能

### 步骤1：授予权限
首次使用时，应用会请求以下权限：
- 📍 **位置权限** - Android系统要求蓝牙扫描需要
- 📡 **蓝牙权限** - 扫描和连接设备
- 🔔 **通知权限** - 前台服务通知

**重要**：请务必全部授予，否则蓝牙功能无法使用。

### 步骤2：打开蓝牙
确保手机的蓝牙已开启。

### 步骤3：扫描设备
1. 打开应用
2. 点击底部导航"**设备**"
3. 点击"**扫描设备**"按钮
4. 等待3-10秒，附近的Hi90B智能戒指会出现在列表中

### 步骤4：连接设备
1. 在扫描到的设备列表中，点击您的设备旁边的"**连接**"按钮
2. 等待连接（通常2-5秒）
3. 连接成功后，设备卡片会显示：
   - 绿色连接指示灯
   - 电量信息
   - 最后同步时间

### 步骤5：查看数据
连接成功后，应用会自动：
- 开始接收实时数据
- 保存到本地数据库
- 更新首页的健康指标

## 🔧 技术实现

### 蓝牙架构
```
智能戒指 (Hi90B)
    ↓ 蓝牙5.0
BleManager (扫描、连接)
    ↓
BleConnectionManager (重连、缓存)
    ↓
DataParser (数据解析)
    ↓
DeviceViewModel (状态管理)
    ↓
DeviceFragment (UI显示)
```

### 数据流向
```
智能戒指传感器
    ↓ BLE GATT
特性通知 (Characteristic Notification)
    ↓
数据解析 (DataParser)
    ↓
传感器数据 (SensorData)
    ↓
数据缓存队列 (Queue)
    ↓
保存到数据库 (Room)
    ↓
UI更新 (LiveData/Flow)
```

## 📊 支持的传感器数据

### 1. 心率 (Heart Rate)
- **采样频率**：1Hz
- **数据范围**：40-200 bpm
- **用途**：睡眠分期、恢复评估

### 2. 血氧 (SpO2)
- **采样频率**：1Hz
- **数据范围**：80-100%
- **用途**：异常检测、睡眠质量

### 3. 体温 (Temperature)
- **采样频率**：0.1Hz (每10秒)
- **数据范围**：32-42°C
- **用途**：体温节律、恢复评估

### 4. PPG原始波形
- **采样频率**：50Hz
- **用途**：HRV计算、深度特征提取

### 5. HRV (心率变异性)
- **采样频率**：实时R-R间期
- **数据范围**：10-200ms
- **用途**：压力评估、恢复指数

### 6. 三轴加速度计
- **采样频率**：25Hz
- **用途**：体动检测、睡眠分期

### 7. 三轴陀螺仪
- **采样频率**：25Hz
- **用途**：姿态识别、运动检测

## 🔄 断线重连机制

### 自动重连策略
1. 检测到断线
2. 等待2秒
3. 尝试重新连接
4. 最多重试3次
5. 失败后提示用户

### 数据缓存
- 断线期间数据缓存到队列
- 重连后继续接收
- 最多缓存1000条数据

## 🐛 常见问题

### Q1：扫描不到设备
**可能原因**：
- 蓝牙未开启
- 位置权限未授予
- 戒指未开机或电量耗尽
- 距离太远

**解决方法**：
1. 检查手机蓝牙是否开启
2. 检查应用权限设置
3. 确保戒指已开机
4. 将戒指靠近手机

### Q2：连接失败
**可能原因**：
- 戒指已连接到其他设备
- 蓝牙信号干扰
- 戒指固件问题

**解决方法**：
1. 断开戒指与其他设备的连接
2. 重启手机蓝牙
3. 重启应用
4. 升级戒指固件

### Q3：频繁断线
**可能原因**：
- 距离过远
- 障碍物阻挡
- 手机蓝牙性能差

**解决方法**：
1. 保持手机和戒指在3米内
2. 避免金属障碍物
3. 关闭其他蓝牙应用

### Q4：数据不准确
**可能原因**：
- 戒指佩戴不正确
- 传感器被污染
- 数据解析错误

**解决方法**：
1. 正确佩戴戒指
2. 清洁传感器
3. 检查数据解析逻辑

### 在ViewModel中使用

```kotlin
class DeviceViewModel(application: Application) : AndroidViewModel(application) {
    private val bleConnectionManager = BleConnectionManager(application)
    
    fun startScan() {
        bleConnectionManager.startScan(callback)
    }
    
    fun connect(deviceAddress: String) {
        bleConnectionManager.connect(

        )
    }
```

### 接收数据

```kotlin
bleConnectionManager.setOnDataReceived { sensorData ->
    // 处理传感器数据
    Log.d("Data", "心率: ${sensorData.heartRate}")
    Log.d("Data", "血氧: ${sensorData.bloodOxygen}")
    
    // 保存到数据库
    saveSensorData(sensorData)
    
    // 异常检测
    if (sensorData.bloodOxygen < 90) {
        triggerAlert("血氧偏低")
    }
}
```
