## 文档维护原则

- 新增 AI 基线文档时，应同步更新 [01_AI阅读入口与文档路径图.md](D:/newstart/docs/00_入口与维护/01_AI阅读入口与文档路径图.md)。
- 当前主上下文基线为 [02_当前项目现状与上下文基线.md](D:/newstart/docs/00_入口与维护/02_当前项目现状与上下文基线.md)，后续架构和功能变更应优先回写该文档。
- 当前材料整理主入口为 [03_当前材料总索引与归档建议.md](D:/newstart/docs/00_入口与维护/03_当前材料总索引与归档建议.md)，新增材料、提交包、证据包或归档策略变化时应优先回写该文档。
- 当运行入口、模块边界、关键接口、认证链路、域名配置发生变化时，应同步更新事实层文档。
- 当统一口径、能力成熟度判断或标准段落发生变化时，应同步更新写作型补充文档。
- 当文档与代码冲突时，以当前代码和当前运行入口为准，并尽快回写文档。
- 默认不删除历史材料，优先通过“降级为历史层”来整理文档体系。
