# 标题
附录 E：核心业务闭环与页面链路

- 适用任务：页面级叙事、业务链路梳理、答辩顺序设计
- 阅读优先级：高
- 是否允许对外直接引用：允许，但必须保留“当前页面入口与数据来源”说明

## 1. 今日页闭环

入口：

- `MorningReportFragment`

读取数据：

- 本地睡眠摘要
- 本地健康指标
- 恢复分
- 建议与解释

输出结果：

- 当日恢复状态
- 与趋势页联动
- 与干预入口联动

价值：

- 它是用户感知“昨天睡得怎么样、今天状态怎样、接下来怎么做”的第一入口

## 2. 设备闭环

入口：

- `DeviceFragment`

动作：

- 扫描设备
- 连接戒指
- 启动采集
- 同步状态到今日页

结果：

- 生理数据进入 Room
- 今日页和趋势页有真实数据源

## 3. 报告闭环

入口：

- `MedicalReportAnalyzeFragment`

动作：

- 上传 PDF / 图片
- OCR 与结构化理解
- 可读报告生成

后续影响：

- 医生问诊可引用报告内容
- 后台报告页可进入分流
- 指标和结论进入患者时间线

## 4. 医生闭环

入口：

- `DoctorChatFragment`

动作：

- 收集主诉、持续时间、伴随症状
- 形成结构化问诊评估
- 生成问诊单

后续影响：

- 与干预中心联动
- 与后台患者详情、时间线联动

## 5. 干预闭环

入口：

- `InterventionCenterFragment`
- `InterventionSessionFragment`
- `RelaxReviewFragment`

动作：

- 选择方案
- 执行干预
- 记录执行数据与 metadata
- 形成复盘

后续影响：

- 趋势页展示执行与效果
- 后台患者工作台可见
- 画像与建议链可继续消费

## 6. 药物 / 饮食闭环

入口：

- `MedicationAnalyzeFragment`
- `FoodAnalyzeFragment`

动作：

- 识别图片
- 生成结构化结果
- 保存记录

后续影响：

- 恢复分解释
- 画像增强
- 后台时间线与建议页

## 7. 呼吸 / Zen / 音景闭环

入口：

- `BreathingCoachFragment`
- `ZenInteractionFragment`
- `InterventionSessionFragment` 的音景会话

当前特点：

- 呼吸训练支持生物反馈与触觉节拍
- Zen 交互支持波浪、手势互动、环境音
- 音景会话支持多轨混音与动态微调

后续影响：

- 执行 metadata 进入本地与云端
- 复盘和趋势可见
- 后续画像和恢复分解释可使用

## 8. 演示闭环

入口：

- 演示账号登录

动作：

- 云端 seed 提前准备数据
- 登录后 bootstrap 灌本地库

结果：

- App 进入可观演状态
- 后台也能按 demo 患者池讲闭环

## 9. 最适合对 AI 描述的方式

后续如果要让 AI 继续写文案，推荐统一使用：

`入口 -> 核心动作 -> 结果 -> 后续影响`

不要只列页面名，也不要只列功能点。
