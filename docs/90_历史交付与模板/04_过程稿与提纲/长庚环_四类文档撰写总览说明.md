## 1. 项目基础信息

### 1.1 项目名称

- 中文名：`长庚环`
- 英文名：`VesperO`
- 团队名：`夜巡者`

### 1.2 项目一句话定义

`长庚环` 是一个以智能戒指连续感知为底座，结合 Android App、云端分析与 AI 助理能力的居家健康管理系统，当前核心目标是把 `监测 -> 分析 -> 建议 -> 执行 -> 反馈` 做成闭环，而不是停留在单纯展示健康数据。

### 1.3 推荐对外表述

面向青年用户和居家健康场景，长庚环通过智能戒指采集睡眠、恢复和生理信号，在移动端提供晨间恢复、症状自查、医检报告理解、AI 问诊和康复辅助，并借助云端完成深度分析与个性化建议生成。

### 1.4 当前最稳的主叙事

当前仓库最适合统一成以下叙事：

- 硬件侧：以智能戒指为核心采集终端，负责连续生理数据获取。
- Android 侧：承接展示、交互、离线兜底、蓝牙连接、问诊和干预执行。
- 云端侧：承接账户、同步、AI 理解、医检报告增强、问诊生成和干预编排。
- 产品目标：从“看到数据”升级为“知道今天应该怎么做”。

### 2.2 Android 侧运行事实

- 当前运行入口是 `:app-shell`，不是 `:app`。

### 2.3 云端事实

- 云端子项目是 `cloud-next/`

- 部署方式：`Vercel`
- 当前生产域名已统一为：
  - `https://cloud.changgengring.cyou/`
- 当前云端职责包括：
  - 认证与用户资料
  - 睡眠/恢复/同步接口

  - 医检报告理解
  - 干预任务/执行/效果接口

  - Avatar 讲解文案路由

### 2.4 共享契约与算法事实

- `contracts/` 已承载跨端契约，采用统一 Envelope：
  - `{ code, message, data, traceId }`
- `ml/` 承载训练与模型导出脚本
- Android 本地仍保留：
  - 本地异常检测
  - 规则回退
  - 本地 OCR 兜底
  - 本地干预草案
- 云端当前已承接：
  - 医生问诊生成
  - 医检报告理解
  - 统一 AI provider 路由

## 3. 产品能力总览

### 3.1 当前主要业务闭环

当前产品最适合总结为两个互相联动的闭环：

#### 闭环 A：戒指驱动的睡眠恢复闭环

`戒指采集 -> 蓝牙同步 -> 睡眠/恢复分析 -> 今日建议/干预 -> 执行记录 -> 趋势反馈`

这一闭环对应的事实基础包括：

- 戒指 BLE 协议、命令构造和数据解析
- 睡眠与恢复相关数据对象
- 晨报/趋势/设备/干预等主页面
- 云端睡眠上传、分析、报告接口

#### 闭环 B：智能问诊与康复辅助闭环

`症状定位/症状选择 -> 风险分级 -> AI 问诊/医检报告理解 -> 康复辅助动作 -> 结果回写`

这一闭环对应的事实基础包括：

- 新的症状自查与辅助诊断向导页面
- 医生问诊页面
- 医检报告 OCR 与 AI 可读摘要
- 干预中心与呼吸/复盘等执行页面

## 4. 戒指、端侧 AI 与云端 AI 的关系

### 4.1 戒指侧

仓库中已经有明确的戒指 BLE 协议实现，说明项目不是纯软件模拟产品，而是基于真实可穿戴终端展开：

- 服务与特征值基于自定义 BLE 协议
- 支持工作参数下发、历史数据量查询、实时传输启停
- 已实现 PPG、心率、血氧、HRV、体温、运动等数据类型解析

### 4.2 端侧能力

Android 本地保留的能力适合在文档中表述为：

- 实时感知与低时延处理
- 弱网/离线兜底
- 蓝牙连接与本地数据持久化
- 本地规则和轻量模型辅助判断

### 4.3 云端能力

云端更适合表述为：

- 统一账号体系与跨设备同步
- 较重的 AI 理解与内容生成
- 报告整理、问诊推理、干预编排
- 管理后台与任务编排

### 4.4 推荐技术表述

如果要写在计划书和开发文档里，建议使用：

`系统采用“智能戒指连续采集 + Android 端边缘处理 + 云端深度分析”的端云协同架构。`

这句话和当前代码事实是对齐的。

## 5. 当前可直接写入比赛文档的技术要点

### 5.1 技术栈

- Android：Kotlin、MVVM、Room、Retrofit/OkHttp、TensorFlow Lite、Nordic BLE、Lottie
- Cloud：Next.js、Supabase、Route Handlers、Vercel
- AI：本地规则/轻量模型 + 云端大模型 provider 路由
- 契约：JSON Schema + Kotlin DTO + TypeScript types

### 5.2 架构亮点

可直接抽取的亮点包括：

- 多模块迁移中的 Android 架构，运行层和业务层分离
- 戒指端、手机端、云端的分层协同
- 本地兜底 + 云端增强的双路 AI 策略
- 认证、云同步、AI 路由、Avatar 讲解、医检理解等统一接入云端

### 5.3 功能亮点

当前代码中可以支撑的功能亮点有：

- 戒指连续感知与 BLE 协议解析
- 今日晨报与恢复评估
- AI 问诊与症状自查
- 医检报告 OCR 与可读化整理
- 康复辅助与执行复盘
- 趋势分析与账户同步

### 5.4 创新点建议口径

当前最稳的创新点写法不是“发明了新算法”，而是以下三类：

- `闭环创新`：从监测走向“可执行建议 + 执行反馈”
- `端云协同创新`：本地实时/隐私兜底，云端负责复杂理解
- `交互创新`：把症状定位、医检报告、AI 问诊和康复动作串成一体化健康助手
