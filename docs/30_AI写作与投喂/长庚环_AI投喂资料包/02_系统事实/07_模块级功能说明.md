# 模块级功能说明

- 适用任务：模块说明、源码导航、模块级文案、功能分工梳理。
- 推荐阅读优先级：高。
- 是否允许对外直接引用：可以转写后引用。
- 最后核对日期：2026-03-15。

## 使用方法

当另一个 AI 需要“某一模块到底负责什么”时，优先看这里。写作时建议采用“模块职责 + 典型入口 + 依赖 + 当前状态”四段式表达。

## `app-shell`

- 模块职责：Android 运行壳、主 Activity、全局导航、桌面机器人、应用初始化。
- 典型入口：`SleepHealthApp`、`MainActivity`。
- 主要依赖：几乎所有 `core-*` 与 `feature-*`。
- 当前状态：已稳定接入。

## `core-common`

- 模块职责：导航图、主题、共享 UI、通用样式、原始资源、图表控件、共用文案资源。
- 典型入口：导航资源、通用卡片、图表样式、字符串资源。
- 主要依赖：被 `app-shell` 与全部 feature 共同依赖。
- 当前状态：已稳定接入。

## `core-model`

- 模块职责：领域模型、协议目录、恢复分模型、生活方式分析模型、干预体验模型。
- 典型入口：`InterventionProtocolCatalog`、日恢复贡献模型、药物/饮食记录模型。
- 主要依赖：被 `core-data` 和多个 feature 消费。
- 当前状态：已稳定接入。

## `core-db`

- 模块职责：Room 数据库、DAO、实体、迁移。
- 典型入口：`AppDatabase` 与各 DAO。
- 主要依赖：`core-data`、部分 ViewModel 初始化。
- 当前状态：已稳定接入。

## `core-data`

- 模块职责：Repository、业务编排、端云协调、DAO 与网络整合、画像与处方汇总。
- 典型入口：`NetworkRepository`、`InterventionProfileRepository`、`PrescriptionRepository`、报告/药物/饮食相关 repository。
- 主要依赖：`core-db`、`core-network`、`core-ml`。
- 当前状态：已稳定接入。

## `core-ble`

- 模块职责：BLE、帧解析、协议和 mock。
- 典型入口：设备扫描与连接、原始数据收发。
- 主要依赖：设备模块与壳层。
- 当前状态：已稳定接入。

## `core-network`

- 模块职责：API 声明、会话存储、token 刷新、第三方网络适配。
- 典型入口：`ApiService`、`ApiClient`、`CloudSessionStore`。
- 主要依赖：`core-data`。
- 当前状态：已稳定接入。

## `core-ml`

- 模块职责：本地模型、本地推理、本地格式化和本地 LLM bridge。
- 典型入口：异常检测、本地 Llama bridge、部分本地报告/解释辅助。
- 主要依赖：`core-data`、部分 feature ViewModel。
- 当前状态：已落地，持续调优。

## `feature-home`

- 模块职责：今日页、晨报、恢复分、建议摘要。
- 典型入口：`MorningReportFragment`。
- 主要依赖：睡眠数据、健康指标、画像与建议聚合。
- 当前状态：已稳定接入。

## `feature-device`

- 模块职责：设备页、扫描连接、设备状态。
- 典型入口：`DeviceFragment`。
- 主要依赖：`core-ble`、设备状态 repository。
- 当前状态：已稳定接入。

## `feature-doctor`

- 模块职责：AI 问诊、问诊历史、结构化问诊单、部分自动讲解。
- 典型入口：`DoctorChatFragment`。
- 主要依赖：doctor turn、结构化文本模型、本地 doctor inference。
- 当前状态：已落地，持续调优。

## `feature-relax`

- 模块职责：干预主模块。
- 典型入口：
  - 症状自查
  - 医检报告
  - 干预中心
  - 干预执行页
  - 呼吸训练
  - Zen 交互
  - 音景干预
  - 药物分析
  - 饮食分析
  - 复盘
- 主要依赖：画像、报告、问诊、实时指标、处方与执行记录。
- 当前状态：已稳定接入，是当前项目最重的业务模块之一。

## `feature-trend`

- 模块职责：周期趋势、周报/月报、执行与恢复趋势。
- 典型入口：`SleepTrendFragment`。
- 主要依赖：睡眠、恢复分、干预执行与周期聚合。
- 当前状态：已落地，持续调优。

## `feature-profile`

- 模块职责：我的、账户设置、通知设置、数据与隐私、关于项目、云账户信息。
- 典型入口：`ProfileFragment` 及设置页。
- 主要依赖：云会话、偏好设置、用户资料。
- 当前状态：已稳定接入。

## 哪些模块最值得在写作里重点提

### 对产品介绍最关键

- `app-shell`
- `feature-home`
- `feature-doctor`
- `feature-relax`
- `feature-trend`

### 对技术架构最关键

- `core-data`
- `core-db`
- `core-network`
- `core-ble`
- `core-ml`

### 对演进说明最关键

- `app-shell`
- `core-common`
- `feature-relax`
- `app/` 历史归档

## 不应误写的点

- 不要把 `feature-relax` 写成“单纯放松页面”。
- 不要把 `core-common` 写成“只有资源文件”。
- 不要把 `feature-profile` 写成“只有个人资料编辑页”。
- 不要把 `app/` 写成当前仍参与构建的主模块。
