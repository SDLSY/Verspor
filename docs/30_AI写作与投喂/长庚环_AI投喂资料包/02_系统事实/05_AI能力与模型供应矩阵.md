# AI能力与模型供应矩阵

- 适用任务：AI 架构介绍、模型清单整理、供应链说明、对外口径控制。
- 推荐阅读优先级：最高。
- 是否允许对外直接引用：可以转写后引用。
- 最后核对日期：2026-03-15。

## 总体原则

当前项目的 AI 不是“单模型单供应商”结构，而是“云端多 provider 编排 + 本地算法兜底 + 部分本地模型”的混合体系。

写作时必须同时说明：

- 哪类能力走云端。
- 哪类能力有本地兜底。
- 哪类能力依赖登录。
- 哪类能力当前仍处于持续调优状态。

## 1. 桌面机器人文案生成

- 作用：根据当前页面内容生成气泡文案和讲解文本。
- 输入：页面 key、标题、副标题、风险摘要、可见重点、触发方式。
- 输出：短文本 + 语义动作。
- 云端主链：`text.structured`
- 当前 provider 顺序：
  - 主：`openrouter`
  - 次：`vector_engine`
  - 兜底：`deepseek`
- 当前默认模型：
  - `openrouter / google/gemini-2.5-flash`
  - `vector_engine / gpt-4.1-mini`
  - `deepseek / deepseek-v3.2`
- 本地兜底：有，本地会在云端文案失败时回退到端侧文本能力。
- 登录依赖：不以用户登录为唯一前提，但依赖云端接口可用。
- 状态标签：已落地，持续调优。

## 2. 医生问诊与问诊单

- 作用：对话式问诊、继续追问、结构化 assessment、问诊单生成。
- 输入：用户对话上下文、症状文本、阶段信息。
- 输出：follow-up 问句或结构化问诊单。
- 云端主链：`text.structured`
- 当前 provider/model 继承同一套文本结构化链：
  - `openrouter / google/gemini-2.5-flash`
  - `vector_engine / gpt-4.1-mini`
  - `deepseek / deepseek-v3.2`
- 本地兜底：有，本地 `DoctorInferenceEngine` 可在云端失败时产出 assessment。
- 登录依赖：通常需要云端可用；用户是否必须登录视具体云会话策略而定，但结构化能力属于云端主链。
- 状态标签：已落地，持续调优。

## 3. 医检报告可读报告与结构化理解

- 作用：把 OCR 结果整理为结构化指标与可读报告。
- 输入：OCR 文本、报告上下文、部分结构化提示。
- 输出：结构化指标、异常解释、可读摘要。
- 云端 provider 顺序：
  - `openrouter`
  - `vector_engine`
  - `deepseek`
- 当前默认模型：
  - `openrouter / google/gemini-2.5-flash`
  - `vector_engine / gpt-4.1-mini`
  - `deepseek / deepseek-v3.2`
- 本地兜底：有，本地会基于 OCR markdown 与格式化逻辑给出退化可读版本。
- 登录依赖：云端解析需要登录会话稳定可用。
- 状态标签：已落地，持续调优。

## 4. 药物图片分析

- 作用：识别药物包装或药品图像，并生成保守型建议。
- 输入：用户上传的图片、必要上下文。
- 输出：候选药名、剂型/规格、成分、风险提示、建议、人工确认标记。
- 云端主 provider：`vector_engine`
- 当前首选模型：`qwen3-vl-235b-a22b-instruct`
- 回退方式：仍在同一 provider 内尝试可用候选模型，而不是切换到完全不同的视觉供应商。
- 本地兜底：未登录时可手动补录并本地保存；不是完全无功能。
- 登录依赖：自动图片解析依赖登录后的云端链路。
- 状态标签：已落地，持续调优。

## 5. 饮食图片分析

- 作用：识别餐食图片并估算热量、营养结构与建议。
- 输入：食物图片、餐次上下文。
- 输出：食物项、热量、三大营养素、营养风险、建议、人工确认标记。
- 云端主 provider：`vector_engine`
- 当前首选模型：`qwen3-vl-235b-a22b-instruct`
- 回退方式：同 provider 内部模型回退。
- 本地兜底：未登录时可手动补录并本地保存。
- 登录依赖：自动图片解析依赖登录后的云端能力。
- 状态标签：已落地，持续调优。

## 6. 语音播报 TTS

- 作用：桌面机器人播报、医生页播报、云端语音合成。
- 输入：文本、voice/profile 等参数。
- 输出：音频 data URL 或可播放音频内容。
- 云端主 provider：`doubao`
- 当前模型与资源：
  - provider：`doubao`
  - resource：`seed-tts-2.0`
  - model：`seed-tts-2.0-standard`
- 当前已兼容的旧 voice 别名：
  - `x4_yezi`
  - `x4_lingxiaoyao_em`
  - `alloy`
- 云端回退：保留 `vector_engine` 音频接口作为次级 fallback。
- Android 端当前真实主链：调用自家云端 TTS 接口，不再默认直连讯飞 TTS。
- 登录依赖：取决于具体接口策略，但 Android 主路径已统一到云端。
- 状态标签：已稳定接入。

## 7. 本地模型与本地算法

### TFLite / 本地检测

- 作用：异常检测、部分健康分析、本地信号处理。
- 输入：本地采集的生理数据或派生特征。
- 输出：异常分数、辅助判断或本地信号结果。
- 云端依赖：否。
- 状态标签：已稳定接入。

### 本地 LLM bridge

- 作用：在部分云端链路失败时作为端侧文本或逻辑兜底。
- 输入：页面文本、问诊上下文或本地解释任务。
- 输出：端侧文本结果或结构化补充。
- 云端依赖：否。
- 状态标签：已落地，持续调优。

## 8. 推荐/处方与解释

- 作用：将画像、风险事实、执行历史综合成干预建议、处方包和解释。
- 输入：量表、问诊、报告、生理指标、生活方式记录、执行历史。
- 输出：任务、建议、处方、推荐解释、效果归因。
- provider：不是固定单模型，而是 cloud 侧推荐与文本结构化链路协同。
- 本地兜底：部分本地画像与规则逻辑存在，但完整推荐闭环仍以云端编排为主。
- 状态标签：已落地，持续调优。

## 不能误写的三件事

1. 不能写成“整个项目只用了一个模型供应商”。
2. 不能写成“所有 AI 能力都必须联网才能运行”。
3. 不能写成“语音播报仍然主走讯飞 TTS”。
