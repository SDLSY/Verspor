- 推荐阅读优先级：最高。

## Cloud 端的一句话事实

`cloud-next` 不是单纯的接口集合，而是一个集成了认证、AI 编排、结构化分析、推荐/处方、后台工作台、模型资产与作业运维的 Next.js 云端系统。

## Cloud 的主要组成

### 前台 API 层

主要位于 `cloud-next/src/app/api`，承接 Android 与后台的云调用，包括：

- 认证与会话：登录、注册、刷新、重置密码。
- 医检报告：OCR、可读报告、结构化解析。
- 医生问诊：turn、assessment、问诊结构化输出。
- 桌面机器人：narration 文案生成。
- 语音：TTS 合成。
- 药物分析与饮食分析。
- 推荐、处方、效果跟踪、内部 worker 等。

### 后台页面层

主要位于 `cloud-next/src/app` 的页面目录下。

当前真正的后台主线是：

- `/dashboard`
- `/patients`
- `/recommendations`
- `/reports`
- `/system`

### 业务逻辑与 AI 编排层

主要位于 `cloud-next/src/lib`：

- `ai/`
- `prescription/`
- `report/`
- `recommendation-model/`
- `model-registry.ts`
- `model-inference.ts`
- `worker-dispatch.ts`
- `admin-*.ts`

### 数据与权限层

- Supabase 负责认证与持久化。
- 后台访问需要 Supabase 登录态 + 管理员限制。
- 关键动作写审计事件。

## 后台工作台的真实主线

### 1. 总览驾驶舱 `dashboard`

作用：

- 汇总高风险患者。
- 汇总最近建议轨迹与效果归因。
- 汇总待处理报告。
- 汇总作业、模型与推荐摘要。

它是运营视角的首页，不是患者自助页面。

### 2. 患者工作台 `patients`

作用：

- 看患者列表。
- 进入单患者详情。
- 串起睡眠、恢复分、量表、问诊、报告、建议轨迹、干预任务、执行记录和时间线。

这是后台最接近“病例/个案工作台”的区域。

### 3. 推荐与处方 `recommendations`

作用：

- 查看建议轨迹。
- 查看效果闭环。
- 配置 SRM_V2 模型资料。
- 管理 recommendation profile。

这部分不是单纯“看推荐结果”，而是能管理配置、版本和解释。

### 4. 报告工作台 `reports`

作用：

- 医检报告汇总。
- 关注高风险、待解析、异常指标与相关摘要。
- 方便人工查看报告理解结果。

### 5. 系统运维 `system`

作用：

- 模型资产管理。
- 作业监控。
- 审计记录。
- 内部 worker 触发。

它是模型/任务/审计的运维层，不是用户功能页。

## 关键 Cloud 业务文件职责

### `admin-dashboard.ts`

- 为后台首页汇总患者、建议、报告、作业等摘要数据。

### `admin-patients.ts`

- 查询患者池与单患者工作台的数据。

### `admin-recommendations.ts`

- 聚合推荐轨迹、效果闭环和配置相关内容。

### `admin-reports.ts`

- 汇总报告视图与待处理项。

### `admin-system.ts`

- 聚合模型、作业、审计和系统级运行视图。

### `admin-mutations.ts`

- 后台改动类操作的主要落点。
- 典型动作会伴随审计记录。

## 云端 AI 编排的角色

`cloud-next` 当前承担的并不只是“转发模型请求”，而是：

- 统一 Provider 选择与回退。
- 做结构化 JSON 约束。
- 做结果归一化。
- 做模型失败后的兜底链路。
- 把结果与业务实体、患者时间线、推荐闭环串起来。

## 端云协同的真实模式

### Android 负责

- 设备接入。
- 本地存储。
- 本地实时反馈与执行体验。
- 部分本地模型兜底。
- 页面呈现与交互。

### Cloud 负责

- 登录态与账号。
- 云端结构化分析。
- 多 provider AI 编排。
- 后台管理与审计。
- 推荐模型配置与作业。
- 云端同步结果。

## cloud-next 不是什么

- 不是一个只给 Android 用的薄网关。
- 不是一个只做 SSR 页面展示的后台壳。
- 不是单 provider、单模型、单用途 AI 服务。

## 对外写作时推荐怎么描述

推荐写法：

- “云端系统负责认证、AI 编排、报告理解、问诊结构化、建议闭环与后台运营工作台。”
- “Cloud 端同时面向 Android 应用与管理后台，承接结构化分析与运维管理。”

不推荐写法：

- “后端只是存数据。”
- “后台只有一个简单管理页。”
- “云端只调用一个大模型。”
