# 关键时序链路

- 适用任务：技术时序说明、答辩讲解、链路级功能介绍、问题定位背景说明。
- 推荐阅读优先级：最高。
- 是否允许对外直接引用：可以转写后引用。
- 最后核对日期：2026-03-15。

## 使用说明

本文件不按页面清单写，而是按“入口 -> 核心动作 -> 结果 -> 后续影响”组织关键链路。给其他 AI 写文案时，优先引用这里的因果关系，而不是只列页面名。

## 1. 戒指连接与实时采集

### 入口

- 用户进入设备页并扫描/连接戒指。

### 核心动作

- `feature-device` 发起扫描与连接。
- `core-ble` 处理 BLE 协议、数据帧、命令交互。
- 健康指标与 PPG 数据写入 Room。
- 连接状态和最新数据供设备页、今日页、干预页等消费。

### 结果

- 本地获得实时心率、血氧、体温、HRV、PPG 等数据。
- 连接状态持续保留，不再局限于设备页可见时。

### 后续影响

- 今日页可刷新生理数据。
- 呼吸训练和音景会话可读最新反馈。
- 趋势页和恢复分可消费最近周期的指标。

## 2. 今日页生理数据刷新

### 入口

- 用户打开“今日”页，或设备连接后返回今日页。

### 核心动作

- `MorningReportViewModel` 加载最近睡眠摘要与最新健康指标。
- 即使当日睡眠记录不完整，也优先显示真实生理指标。
- 本地聚合恢复分、建议和说明项。

### 结果

- 用户看到最新心率、血氧、体温、HRV 等摘要。
- 晨报、恢复分和建议卡片同步更新。

### 后续影响

- 今日页成为用户每天查看恢复状态与建议的总入口。

## 3. 医检报告 OCR -> 可读报告

### 入口

- 用户在医检报告页上传 PDF 或图像。

### 核心动作

- 本地先完成原始文件导入与基础解析。
- 云端 `report-understanding` 对 OCR 文本和上下文做结构化理解。
- 云端尝试输出可读报告、结构化指标、风险摘要。
- 若云端失败，本地仍会用 OCR markdown 与格式化逻辑给出降级可读版本。

### 结果

- 用户得到更易读的报告解释，而不是只看原始 OCR。
- 本地与云端都保留结构化结果和状态。

### 后续影响

- 报告异常进入画像和风险事实。
- 后台可在报告工作台查看待处理与高风险项。

## 4. 医生问诊 -> 结构化问诊单

### 入口

- 用户在医生页与 AI 进行问诊对话，随后生成问诊单。

### 核心动作

- Android 发送用户当前对话上下文到云端 doctor turn。
- 云端使用结构化文本模型输出 follow-up 或 assessment。
- 当阶段要求直接生成 assessment 时，云端和本地引擎都保证返回问诊单而不是继续追问。
- 如果云端不稳定，本地 doctor inference engine 会兜底产出结构化问诊单。

### 结果

- 页面展示问诊总结、疑似问题、风险提醒、下一步建议等。

### 后续影响

- 问诊单进入患者时间线和本地历史。
- 画像与建议上下文可消费问诊结论。

## 5. 干预生成 -> 执行 -> 回写 -> 趋势

### 入口

- 用户从干预中心、医生页或今日建议进入干预内容。

### 核心动作

- 画像与处方上下文生成任务和建议项。
- 用户启动呼吸训练、音景会话、Zen 交互等具体执行。
- 执行过程中采集时长、状态、效果、metadata。
- 完成后写 `relax_sessions` 和 `intervention_executions`。
- 记录再上云或参与本地趋势聚合。

### 结果

- 干预不再是一次性页面动作，而是可统计、可回顾、可纳入恢复分的执行事件。

### 后续影响

- 复盘页可展示效果。
- 趋势页可展示执行次数与模态效果。
- 恢复分可纳入最近 24 小时干预贡献。

## 6. 桌面机器人点击 -> 文案生成 -> TTS 播报

### 入口

- 用户点击桌面机器人，或进入某些页面触发入场讲解。

### 核心动作

- `MainActivity` 收集当前页面 key、标题、摘要、可见重点。
- 云端 `/api/avatar/narration` 生成适合气泡与播报的短文案。
- Android 展示文字气泡。
- TTS 通过 `/api/ai/speech/synthesize` 生成音频。
- 云端 TTS 主路径当前为豆包，失败时可回退到保留链路。

### 结果

- 用户获得页面导览、风险提示、操作建议。

### 后续影响

- 桌面机器人承担“跨页面解释层”，而不是单页组件。

## 7. 药物/饮食图片分析 -> 画像/恢复分

### 入口

- 用户在药物分析或饮食分析页拍照或导入图片。

### 核心动作

- 已登录时，云端结构化图片分析直接解析图像。
- 未登录时，用户可手动补录并本地保存。
- 药物与饮食记录写本地表，并携带 provider/model/trace/syncState。
- 记录进入画像、建议上下文和恢复分贡献项。

### 结果

- 用户获得识别结果、风险提示、营养或药物建议。

### 后续影响

- 画像中增加生活方式证据。
- 晨报与趋势开始利用这部分信息。
- 后台患者时间线也能消费这类事件。

## 8. 趋势页周报/月报

### 入口

- 用户在趋势页切换近 7 天或近 30 天。

### 核心动作

- ViewModel 根据时间范围重建报告标题与摘要。
- 聚合睡眠、恢复分、干预执行等周期数据。
- 卡片标题和页标题都以前端当前周期为准，而不是直接沿用后端旧标题。

### 结果

- 近 7 天展示趋势周报。
- 近 30 天展示趋势月报。

### 后续影响

- 趋势页成为恢复趋势与干预效果的周期回顾页。

## 9. 新放松能力链路

### 呼吸训练

- 入口：干预中心或医生建议。
- 核心动作：呼吸节律 + 实时生理反馈 + 触觉节拍。
- 结果：生成带 metadata 的放松会话。
- 后续影响：写入恢复分与复盘。

### 音景干预

- 入口：音景协议，如 `SOUNDSCAPE_SLEEP_AUDIO_15M`。
- 核心动作：多轨混音 + 自适应微调 + 手动调节。
- 结果：形成音景型干预执行记录。
- 后续影响：进入趋势与画像。

### Zen 轻交互

- 入口：`ZEN_MIST_ERASE_5M`、`ZEN_WAVE_GARDEN_5M`。
- 核心动作：本地画布、触摸交互、环境音与会话记录。
- 结果：形成轻交互型干预记录。
- 后续影响：进入执行统计和次日恢复贡献。

## 给别的 AI 的一句提醒

描述本项目时，最好把这些链路写成“输入、分析、执行、回写”的闭环，而不是拆成互不关联的功能点。
