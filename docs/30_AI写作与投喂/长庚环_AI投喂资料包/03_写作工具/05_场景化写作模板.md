# 场景化写作模板

- 适用任务：官网文案、比赛摘要、技术文档、测试文档、答辩讲稿、模块级文案。
- 推荐阅读优先级：高。
- 是否允许对外直接引用：内部使用为主。
- 最后核对日期：2026-03-15。

## 1. 官网/宣传文案模板

```text
请基于资料包，写一版面向普通用户和合作方的官网式项目介绍。

要求：
1. 先用 1 句话定义长庚环。
2. 用 3 段说明：
   - 它解决什么问题
   - 它如何形成“感知 -> 分析 -> 干预 -> 反馈”的闭环
   - 它与普通睡眠 App 的区别
3. 不要使用“确诊、治疗、自动处方”等医疗越界表述。
4. 允许强调创新性，但不能夸大为完全商用医疗系统。
5. 语言要偏产品化、易理解。
```

## 2. 比赛技术研究报告模板

```text
请基于资料包，写一版适合比赛技术报告的项目说明。

要求：
1. 先概括项目定位与研究意义。
2. 再写系统架构，包括 Android、Cloud、AI 编排、硬件采集、数据闭环。
3. 明确说明端云协同、本地兜底、多 provider 的价值。
4. 对每类 AI 能力写清输入、输出、落点和边界。
5. 对“已落地，持续调优”和“研究/规划中”能力做状态区分。
6. 风格偏技术研究，不要写成营销文案。
```

## 3. 项目开发文档模板

```text
请基于资料包，写一版给新开发者的项目开发说明。

要求：
1. 先说明当前 Android 入口、模块边界与 cloud-next 的角色。
2. 再解释关键数据实体、DAO、repository、时序链路。
3. 明确 app/ 是历史归档，不是当前运行入口。
4. 给出各 feature 的职责与典型入口。
5. 风格偏工程导向，少宣传，多事实。
```

## 4. 测试文档模板

```text
请基于资料包，写一版测试视角的系统说明。

要求：
1. 按关键闭环拆测试范围：
   - 戒指连接与采集
   - 今日页恢复分与生理数据
   - 医生问诊
   - 医检报告
   - 干预执行
   - 药物/饮食分析
   - 桌面机器人与 TTS
2. 每条闭环写入口、依赖、关键结果、失败回退。
3. 明确哪些能力依赖登录、云端或本地兜底。
```

## 5. 答辩讲稿模板

```text
请基于资料包，写一版 5 分钟答辩讲稿。

要求：
1. 先讲痛点，再讲方案，再讲闭环，再讲创新点，再讲边界。
2. 创新点必须落到源码事实，不要空泛。
3. 强调：
   - 硬件感知不是摆设
   - Android 端不是纯展示层
   - 云端不是纯存储层
   - 干预不是一次性内容播放
4. 风格要有节奏感，但不能夸大为医疗确诊系统。
```

## 6. 模块级页面文案模板

```text
请基于资料包，为指定页面撰写模块级说明文案。

要求：
1. 先说明页面在整体闭环里的位置。
2. 再说明页面的核心输入、核心动作、输出结果和后续影响。
3. 如果页面涉及 AI 能力，必须说明它属于辅助理解/辅助判断/建议生成，不要写成医疗确诊。
4. 如果页面属于持续调优能力，措辞要克制。
```
