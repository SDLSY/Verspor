# 十分钟项目全景

- 适用任务：给另一个 AI 做极速预热。
- 推荐阅读优先级：最高。
- 是否允许对外直接引用：可以转写，不建议原样引用。
- 最后核对日期：2026-03-15。

## 一句话定义

长庚环是一个围绕“戒指连续感知 + Android 单入口交互 + 云端 AI 增强 + 干预执行回写”构建的睡眠恢复与健康辅助系统。

## 三层架构

### 1. 感知与本地执行层

- 智能戒指通过 BLE 向 Android 端提供心率、血氧、体温、PPG、运动等数据。
- Android 端负责连接、采集、落库、页面交互、部分本地模型推理、本地兜底和干预执行。
- 当前唯一 Android 运行入口是 `:app-shell`。

### 2. 业务编排层

- Android 通过 `core-*` 和 `feature-*` 模块组织业务。
- 主业务不是“看一堆指标”，而是“统一画像 -> 处方/建议 -> 执行 -> 反馈”。
- 今日页、医生页、趋势页、设备页、我的页是主导航，症状自查、医检报告、干预中心、呼吸训练、Zen 交互、音景会话等是二级链路。

### 3. 云端增强层

- `cloud-next` 提供认证、报告理解、医生问诊增强、推荐解释、干预任务回写、后台工作台、AI 路由和管理接口。
- `contracts` 提供跨 Android / Cloud / ML 的共享契约。
- `ml` 提供训练、导出、验证和推理脚本，不是当前主运行入口。

## 当前产品主线

可以稳定使用的主线不是单一“睡眠报告”，而是：

1. 戒指或用户输入提供状态信息
2. Android 汇总并展示今日状态
3. 云端或本地 AI 生成辅助理解与建议
4. 用户进入干预中心、呼吸训练、Zen 交互、音景等执行页
5. 执行结果回写到本地/云端
6. 次日恢复分和趋势页继续反映执行效果

## 当前已经明确落地的能力

- Android 单入口运行
- BLE 设备扫描、连接、前台采集、后台采集服务
- 今日页、医生页、趋势页、设备页、我的页
- 症状自查与辅助判断路径
- 医检报告 OCR / 结构化 / 可读报告
- AI 医生问诊与结构化问诊单
- 干预中心、呼吸训练、Zen 轻交互、音景干预
- 桌面机器人文案生成与 TTS 播报
- 药物图片分析、食物图片分析接入画像与恢复分
- 云端后台工作台与推荐/患者/报告/系统管理页

## 必须保守表达的边界

- 长庚环是健康辅助系统，不是正式医疗诊断平台。
- 症状自查、AI 医生、报告理解、药物建议都属于辅助理解与风险提示，不等于医疗结论。
- 模块化工程已经成型，但不能写成“所有历史代码已彻底消除”；`app/` 仍作为 legacy 归档存在。
- 不是所有 AI 能力都走同一供应商，必须按能力区分 provider 与模型。

## 当前最容易写错的点

1. 把 `app/` 旧单体写成当前运行入口。
2. 把长庚环写成只有睡眠监测，没有医生、报告和干预闭环。
3. 把 AI 功能写成正式诊断、处方或治疗系统。
4. 把所有云端文本能力、视觉能力和 TTS 写成一个模型供应商。
5. 忽略桌面机器人、执行回写、恢复分和趋势反馈的闭环作用。
