# 标准投喂提示词模板

- 适用任务：把本资料包投喂给其他 AI，生成新的项目介绍、技术文案、比赛材料或模块文案。
- 推荐阅读优先级：最高。
- 是否允许对外直接引用：内部使用为主。
- 最后核对日期：2026-03-15。

## 通用投喂模板

把下面这段作为其他 AI 的起始 prompt：

```text
你将基于一套“源码对齐最终版”的项目资料包来理解长庚环项目，并继续完成指定写作任务。

请先严格遵守以下规则：
1. 只以资料包中的事实为准，不要自行脑补 README、历史比赛文稿或常见健康 App 设定。
2. 在开始正文前，先输出四项内容：
   - 项目一句话定义
   - 三层架构摘要
   - 核心闭环路径
   - 五条最容易误写的边界
3. 明确区分以下状态标签：
   - 已稳定接入
   - 已落地，持续调优
   - 研究/规划中
   - 历史归档
4. 涉及医生问诊、医检报告、药物分析、饮食分析时，必须写成“辅助理解、辅助判断、风险提示、健康建议”，不能写成正式医疗诊断或自动处方。
5. Android 当前唯一运行入口是 app-shell，app/ 是历史归档，不能写反。
6. 项目不是单一睡眠 App，而是“戒指感知 + Android 单入口 + 云端 AI + 干预闭环”的健康辅助系统。
7. 项目是多 provider、端云协同、含本地兜底的 AI 架构，不能写成单模型单供应商。

在你完成四项前置总结后，再根据我的任务开始正式写作。
```

## 推荐附加说明

如果你要它写得更稳，可以再追加：

```text
如果你发现资料包中对某能力的状态是“已落地，持续调优”，正文中不要把它写成完全成熟商用能力。
如果你发现某条能力属于“历史归档”，不要把它写成当前现行入口或核心事实。
```

## 如果目标是更技术化

可以额外加：

```text
请优先引用架构、模块边界、时序链路和 AI 供应矩阵，不要只按页面功能列表组织正文。
```

## 如果目标是更产品化

可以额外加：

```text
请优先用“用户入口 -> 核心动作 -> 结果 -> 后续影响”的方式组织叙述，而不是直接罗列页面。
```

## 投喂后的理想第一屏

理想情况下，另一个 AI 第一段应该先输出：

- 长庚环是什么
- 三层架构是什么
- 闭环是什么
- 哪些边界不能夸大

如果它一上来就直接开始写“这是一款睡眠监测 App”，说明它没有正确吃透资料包。
