# 测试证据目录

本目录是当前仓库的正式测试证据层，用于保存测试准备、单元测试、功能测试、系统测试以及按 case 编号整理的截图、日志、表格和汇总材料。

## 当前结构

- `01-plan/`
  - 测试准备、环境、设备、版本和依赖快照
- `02-unit/`
  - 核心模块单元测试证据
- `03-functional/`
  - 业务闭环功能测试证据
- `04-system/`
  - 系统联动、稳定性与性能测试证据
- `cases/`
  - 按测试编号拆分的个案证据
- `summary/`
  - 汇总索引、缺陷单和章节草稿

## 使用规则

- 新增正式测试材料时，优先放入本目录，不再散落到仓库根目录。
- 同一条测试链路的截图、日志、表格和结论应尽量放在同一 case 或同一组测试目录下。
- 若只是开发过程中的临时验证，不应直接放进本目录，应先进入 `artifacts/`，确认需要保留后再提升到这里。
