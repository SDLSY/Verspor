## 🏗️ 技术架构详解

### 1. 模型架构

#### 整体结构
```
输入 (7种模态)
    ↓
├─ PPGEncoder (1D-CNN)          → 64维
├─ HR Encoder (MLP)             → 32维  
├─ HRV Encoder (MLP)            → 32维
├─ SpO2 Encoder (MLP)           → 16维
├─ Temp Encoder (MLP)           → 16维
└─ Motion Encoder (Bi-LSTM)     → 32维
    ↓
特征拼接 (192维)
    ↓
投影层 (192→176维)
    ↓
位置编码 (Positional Encoding)
    ↓
Transformer Encoder (4层, 8头)
    ↓
多任务输出
├─ 睡眠分期 (5类分类)
├─ 恢复指数 (回归, 0-100)
└─ 异常检测 (二分类)
```

#### 参数量统计
- **总参数**: 约280万 (2.8M)
- **可训练参数**: 约280万
- **FP32模型大小**: 约45MB
- **FP16模型大小**: 约23MB
- **INT8模型大小**: 约12MB

### 2. 各模态编码器设计

#### PPG编码器（CNN）
- **输入**: `[batch, 1, 1500]` (30秒×50Hz)
- **架构**: 3层Conv1d + BatchNorm + ReLU + Dropout
- **输出**: `[batch, 64]`
- **特点**: 提取PPG波形的局部模式特征

#### 运动编码器（Bi-LSTM）
- **输入**: `[batch, 750, 6]` (30秒×25Hz, 6轴传感器)
- **架构**: 2层双向LSTM
- **输出**: `[batch, 32]`
- **特点**: 捕获运动的时序依赖性

#### 统计特征编码器（MLP）
- **输入**: 各种统计特征向量
- **架构**: 2层全连接 + BatchNorm
- **输出**: 固定维度向量
- **特点**: 简单高效，适合低维特征

### 3. Transformer融合层

- **d_model**: 176 (模型维度)
- **nhead**: 8 (注意力头数)
- **num_layers**: 4 (Transformer层数)
- **dim_feedforward**: 512 (前馈网络维度)
- **dropout**: 0.1
- **激活函数**: GELU (优于ReLU)

**关键特性**:
- 多头自注意力：捕获不同方面的模态间依赖
- 位置编码：为时序数据添加位置信息
- Layer Normalization：稳定训练
- 残差连接：缓解梯度消失

### 4. 多任务学习

#### 任务1：睡眠分期（分类）
- **输出**: 5类 (清醒, N1, N2, N3, REM)
- **损失**: CrossEntropyLoss
- **评估**: Accuracy, Kappa系数
- **权重**: α = 1.0

#### 任务2：恢复指数（回归）
- **输出**: 1个值 (0-100分)
- **损失**: MSELoss
- **评估**: MAE, RMSE, R²
- **权重**: β = 0.5

#### 任务3：异常检测（二分类）
- **输出**: 1个值 (0-1概率)
- **损失**: BCELoss
- **评估**: Precision, Recall, F1
- **权重**: γ = 0.3

**总损失**: `L = α·L_stage + β·L_recovery + γ·L_anomaly`

## 🔬 数据预处理流程

### 1. 数据清洗
```python
方法: IQR (四分位距)
阈值: 3倍IQR
处理: 异常值替换为中位数
```

### 2. 信号滤波

| 信号类型 | 滤波方法 | 参数 |
|---------|---------|------|
| PPG | 带通滤波 | 0.5-5Hz, 4阶Butterworth |
| 加速度 | 低通滤波 | 5Hz截止, 4阶Butterworth |
| 心率 | 中值滤波 | 窗口=5 |

### 3. 特征提取

#### 心率特征 (4维)
- 均值、标准差、最小值、最大值

#### HRV特征 (3维)
- RMSSD (时域)
- SDNN (时域)
- LF/HF比率 (频域)

#### 血氧特征 (3维)
- 均值、标准差、最小值

#### 体温特征 (2维)
- 均值、趋势 (线性回归斜率)

### 4. 窗口划分
- **窗口大小**: 30秒
- **步长**: 15秒 (50%重叠)
- **输出**: 多个30秒窗口样本

## 🎯 训练流程

### 1. 数据划分
- **训练集**: 70%
- **验证集**: 15%
- **测试集**: 15%

### 2. 训练超参数
```python
batch_size = 64
epochs = 100
learning_rate = 1e-4
optimizer = AdamW (weight_decay=0.01)
```

## 📤 模型导出

### 1. 导出格式

| 格式 | 用途 | 工具 |
|-----|------|------|
| PyTorch (.pth) | 训练/微调 | torch.save |
| ONNX (.onnx) | 跨平台推理 | torch.onnx |
| TFLite (.tflite) | Android端侧 | TensorFlow Lite |

### 2. 量化策略

| 量化类型 | 大小 | 速度 | 精度 | 适用场景 |
|---------|------|------|------|---------|
| FP32 | 45MB | 基准 | 100% | 云端推理 |
| FP16 | 23MB | 1.5x | 98% | 端侧推理(推荐) |
| INT8 | 12MB | 2.5x | 94% | 端侧异常检测 |

### 3. 导出命令
```bash
# 导出ONNX
python export_model.py --checkpoint ./checkpoints/best_model.pth

# 生成所有格式（包括量化版本）
python export_model.py \
    --checkpoint ./checkpoints/best_model.pth \
    --output_dir ./exported_models
```

## 🎓 技术创新点

### 1. 多模态融合
- **创新**: 业界首次融合7种生理信号
- **方法**: 各模态独立编码 + Transformer融合
- **效果**: 准确率提升5-10%

### 2. 端云协同
- **创新**: 动态卸载策略
- **方法**: 端侧轻量模型 + 云端重型模型
- **效果**: 平衡实时性与准确性

### 3. 多任务学习
- **创新**: 同时优化3个相关任务
- **方法**: 共享底层表示 + 任务特定头
- **效果**: 提升泛化能力

### 4. 可解释性
- **创新**: 提供归因分析
- **方法**: SHAP值量化各因素贡献
- **效果**: 用户信任度提升
